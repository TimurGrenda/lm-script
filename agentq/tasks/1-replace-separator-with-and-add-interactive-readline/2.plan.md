# Task 2: Add get_readline_input() and wire up main()

**Plan**: 1-replace-separator-with-and-add-interactive-readline
**Status**: Pending
**Depends on**: Task 1

## Goal

Add a function for interactive readline prompt input and update `main()` to use it when `INTERACTIVE_PROMPT` is true.

## Context

The `lm` script has two existing input methods:
1. Inline prompt via `$PROMPT` variable (set by parse_args when text follows the separator)
2. Editor-based input via `get_input()` function (opens `$EDITOR`)

We need a third: interactive readline input that reads a single line from the terminal.

The script already has `LM_TTY` (line 10) for controlling tty I/O in tests:
```bash
LM_TTY="${LM_TTY:-/dev/tty}"
```

Current main() logic (lines 133-142):
```bash
if [[ -n "$PROMPT" ]]; then
  run_llm "${LLM_OPTS[@]}" "$PROMPT"
else
  local input
  input="$(get_input)"
  [ -z "$input" ] && exit 0
  run_llm "${LLM_OPTS[@]}" "$input"
fi
```

## Implementation

**File**: `lm`
**Action**: modify

1. Add `get_readline_input()` function after `run_llm()` (after line 20):
   ```bash
   # Read a single-line prompt interactively via readline.
   # Used when ":" is the last argument, bypassing shell metacharacter issues.
   get_readline_input() {
     local input
     # -e: use readline for line editing
     # -r: do not interpret backslash escapes
     read -e -r -p "prompt: " input <"$LM_TTY" >"$LM_TTY"
     echo "$input"
   }
   ```

2. Update main() to handle the three input modes. Replace lines 133-142 with:
   ```bash
   if [[ -n "$PROMPT" ]]; then
     run_llm "${LLM_OPTS[@]}" "$PROMPT"
   elif [[ "$INTERACTIVE_PROMPT" == true ]]; then
     local input
     input="$(get_readline_input)"
     [ -z "$input" ] && exit 0
     run_llm "${LLM_OPTS[@]}" "$input"
   else
     local input
     input="$(get_input)"
     [ -z "$input" ] && exit 0
     run_llm "${LLM_OPTS[@]}" "$input"
   fi
   ```
