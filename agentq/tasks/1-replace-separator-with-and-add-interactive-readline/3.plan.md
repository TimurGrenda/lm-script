For the interactive mode integration test, we need to feed input to the script. Since `LM_TTY` is set to `/dev/null` in tests, we need to handle this. One approach: create a temp file with the prompt text, set `LM_TTY` to that file for the specific test. Or better: use a named pipe / process substitution. Simplest: write the prompt to a file and redirect `LM_TTY` to it.

```bash
@test "interactive readline mode passes prompt to llm" {
  # Create a file with prompt text to feed to readline
  local prompt_file
  prompt_file="$(mktemp)"
  echo "explain bash" > "$prompt_file"
  export LM_TTY="$prompt_file"

  run "$LM_SCRIPT" :

  assert_success
  run cat "$MOCK_LLM_LOG"
  assert_output --partial "explain bash"

  rm -f "$prompt_file"
}
```
